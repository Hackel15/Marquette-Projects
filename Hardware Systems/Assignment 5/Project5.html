Preparation

You will have to familiarize yourself with several common UNIX tools for this assignment. The first of these is tar, a utility originally devised to create tape archives for the purpose of backing files up onto computer tapes.

While tar is still used to create tape backups of filesystems, it has become far more common to use tar to group files and/or directories together into a single entity, typically called a "tar-ball". (So common is the use of tar that it has been verbed in computer science terminology: We speak of "tarring" files, or files that have been "tarred up".) Tar syntax is somewhat arcane, as tar came into existence before modern standards for command-line options.

  • Login to Morbius, and change to a fresh working directory.
  • Execute the the following command:
    tar xvzf ~brylow/cosc2200/armPlayground.tgz

This untars the files into your working directory, in a subdirectory called armPlayground/. For more information on tar, please see the UNIX man pages.

Edit the file main.S to write your assembler programs. Your work must be compiled on a machine with the proper tools, such as the Systems Lab machines. In order to compile your program, use the command "make". In order to run your program, use the command "arm-console". At any time, you can shutdown the remote console system by hitting Ctrl-Space, followed by the letter 'q', for 'quit'.

Few rules govern the format of assembly language programs. Make an effort to keep your programs readable and well-documented; sometimes partial credit may be given if we can tell what you were trying to do, even if it doesn't quite make it.

The ARM Playground kernel includes several useful functions for input and output in these earliest assignments. The first is getnum. When executing the getnum function, (with the assembly code " bl getnum",) the user is prompted with a question mark, and a simple integer will be read into register r0. The printnum function will output a base-10 representation of the contents of register r0. The divide function takes a dividend in r0 and a divisor in r1; upon return, the integer quotient is stored in r0 and any remainder in r1.

In keeping with the standard ARM calling convention, registers r0 through r3 may have their values changed by any function call.

Q1 - Kelvin

Write an assembler program that reads an integer, and converts that integer from Fahrenheit to Kelvin. You need only use integers for the conversion, not floating point numbers.

Example:
? 50
283

You may assume that overflow and underflow do NOT occur.

Q2 - Factorial

Write an assembler program that reads one integer and prints the factorial of that integer.

Example:
? 10
3628800

You may assume that overflow and underflow do NOT occur, but factorials grow quickly and your program will not return correct answers past the billions range.  (Why?)

Q3 - Counts

Write an assembler program that reads in integers from the user until a zero is entered. Print the sum of the integers, the product of multiplying all of the integers, the number of even integers seen, and the number of odd integers seen.  Do not include the 0 in your calculations.

Example:
? 3
? 4
? 5
? 0
12
60
1
2

You may assume that overflow and underflow do NOT occur.
