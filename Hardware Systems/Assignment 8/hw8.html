Q1 - The Towers of Hanoi Problem

Write an ARM Assembly program that asks the user to input the number of pegs and then prints the steps needed in order to solve the Tower of Hanoi problem instance. You should implement a recursive solution. If you are not familiar with the Towers of Hanoi problem and the recursive algorithm to solve it, there are many resources on the web about it including this YouTube video.

We have provided an example program for your reference, runnable on Morbius as ~debbie/public/cosc2200/hanoi

Q2 - Heap Limits

Write an ARM Assembly language program that requests dynamic allocation of memory from the operating system (malloc function) until the operating system will not allocate any more.  Submit your written answers to the D2L Dropbox.

Analysis Questions:

  • How much memory will the ARM Playground give you?
  • Experiment with different request sizes. What do you see?
  • What is the lowest address returned by malloc? What is the highest?

Q3 - Text Dump

Write an ARM Assembly program that prints out the memory address and the contents of every location between "main:" and the last line of your main program. Recall that the "%X" format specifier to kprintf() prints hexadecimal values.

Analysis Questions:

  • Where is your program in memory?
  • Does your program change locations if you run it again?
  • What is the meaning of what you are displaying?

Submit your written answers to the D2L dropbox.

Q4 - Nuke Stack

Having worked so hard to understand the discipline of activation records, now let's break them. Write ARM Assembly function "nukeStack" that when called -- from anywhere in a program that obeys standard calling conventions-- destroys all of the activation records above it on the stack until it finds the stack frame for the main program, and then returns directly to the main program. Suffice it to say, nukeStack does not follow the standard calling convention. Test your nukeStack by calling it from within various programs from previous assignments.
